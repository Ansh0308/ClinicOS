Design Principles

01
Rounded Over Sharp
Every element — buttons, cards, inputs, avatars, even the page itself — has generous border-radius. Minimum 12px; maximum 100px (pill). The roundness communicates safety, friendliness, and approachability. Sharp corners are a last resort for data tables only.
02
Warm Backgrounds, Never Cold
The cream palette (#F5EDD8, #EDE3C8) is non-negotiable. Never use grey or pure white as a main background. The warmth is what differentiates ClinicOS from generic SaaS dashboards. It also reduces eye strain in clinic environments where screens are on all day.
03
Depth Through Layering
The signature layered header with progressively darker bands behind the main nav creates physical depth. This principle extends to all overlapping elements: cards cast subtle shadows, modals lift from the page, dropdowns float. Never flat when depth adds meaning.
04
Fredoka for Headlines, Always
The brand personality lives in the display font. Every headline, number, token badge, and logo uses Fredoka. It must never be mixed with another display font. DM Sans handles all UI copy and body text — these two fonts must never cross their domains.
05
One Strong CTA Per Screen
Following the reference site exactly — there is one primary action per screen. On dark backgrounds it's the yellow button. On light backgrounds it's the dark maroon button. Multiple primary CTAs compete and cancel each other. One wins; the rest are secondary or ghost.
06
Numbers Are Heroes
In clinic dashboards, numbers matter most. Patients seen: 47. Revenue: ₹12,450. Wait time: 23 mins. These numbers should be displayed in Fredoka at the largest possible size, with supporting context in small DM Sans below. Make the number the first thing the eye lands on.

Do & Don't

✓ Do
Use warm cream backgrounds (#F5EDD8) as the default page surface

Use Fredoka for all headings, numbers, and the logo

Round everything generously — minimum 12px radius on any card or input

Use the layered header effect on the navigation at all times

Keep section labels as outlined pills above every major heading

Use the yellow CTA (#F0C030) for the single most important action on dark sections

Give cards generous shadow and clear visual hierarchy inside them

Show phone mockups of the app UI in hero and feature sections
✗ Don't
Use grey or cool-tone backgrounds — they break the warm, caring aesthetic

Use Inter, Roboto, or system fonts for headlines — only Fredoka

Use sharp corners (0px radius) on any interactive element or card

Use purple gradients on white — generic SaaS look that conflicts with this brand

Use more than one primary CTA on the same screen section

Mix too many accent colors in one section — max 2 accent colors per component

Use clinical blue as the primary brand color — reserved for info states only

Display small body text on the crimson background — maintain 4.5:1 contrast ratio

ClinicOS-Specific Design Decisions

The Pomegranate Health reference is a consumer health app. ClinicOS is a B2B2C clinic management platform. These adaptations are required to apply the visual language to the ClinicOS context correctly.
Design DecisionPomegranate ReferenceClinicOS Adaptation
Primary audience screenConsumer mobile appStaff desktop/tablet dashboard is the primary screen — must work at 1280px+
Data densityLow — one metric per cardHigher — queue rows, patient lists, stats grids are all dense; maintain breathing room within each row
Color on statusUsed decorativelyMust carry semantic meaning: crimson = active/urgent, yellow = next/warning, teal = done/healthy, gray = waiting
Font size minimumConsumer: comfortable sizingClinic: 14px body minimum — rooms are sometimes dim; screens may be viewed at distance
Button labelingBrand-focused ("Join Waiting List")Action-focused ("Issue Token", "Mark Served", "Generate Bill") — clarity over cleverness
NavigationSingle global navRole-based portals: Patient / Doctor / Staff / Admin each have their own nav context
Crimson usageDecorative, brandCrimson = also emergency/urgent state; use sparingly in non-emergency dashboard contexts to preserve signal value
AnimationUsed freely for delightLimit to transitions < 200ms for queue updates; heavy animations distract staff during patient management

Role-Based Color Accents

Doctor Teal gradient Doctor portal, consultation state
Admin / Analytics Lavender gradient Admin portal, chart accent
